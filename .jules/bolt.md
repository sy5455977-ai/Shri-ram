## 2026-04-29 - Optimizing List Re-renders and Callback Stability
**Learning:** In React applications with long lists (chat messages, conversation history), passing global state IDs (like `copiedId` or `activeConversationId`) or list indices to memoized children causes O(n) re-renders when those values change. Additionally, callbacks that depend on the list state itself (like `handleRegenerate`) will be re-created on every list update, triggering unnecessary re-renders of all memoized children that receive them.
**Action:** Use derived boolean flags (e.g., `isCopied`, `isActive`, `isLast`) instead of raw IDs or indices. Stabilize callbacks by using a `useRef` to track the list state and referencing that ref within `useCallback` to keep the function identity stable across list updates.
